服薬管理コマンド

/med
服薬記録を追加

指定したユーザーが薬を飲んだことを記録します。記録後、更新されたカレンダーが表示されます。

user 必須

服薬を記録する対象のユーザーを @メンション で指定します。

/med @ユーザー名

💡 使用例

ユーザー「田中さん」の服薬記録を追加する場合：

/med @田中さん

📋 レスポンス例

「田中さん、よくできました！」というメッセージと共に、更新されたカレンダーが表示されます。

/check
服薬状況を確認

指定したユーザーの現在の服薬カレンダーを表示します。その日の服薬状況に応じたメッセージも表示されます。

user 必須

状況を確認したいユーザーを @メンション で指定します。

/check @ユーザー名

💡 使用例

ユーザー「佐藤さん」の服薬状況を確認する場合：

/check @佐藤さん

📋 レスポンス例

服薬済みの場合：「佐藤さんは今日のお薬はもう飲みましたね！えらい！」
未服薬の場合：「佐藤さんは今日のお薬はまだのようです。忘れないでくださいね。」

/uncheck
服薬記録を取り消し

指定したユーザーの今日の服薬記録を取り消します。間違えて記録してしまった場合に使用します。

user 必須

記録を取り消したいユーザーを @メンション で指定します。

/uncheck @ユーザー名

💡 使用例

ユーザー「山田さん」の今日の服薬記録を取り消す場合：

/uncheck @山田さん
⚠️ 注意: このコマンドは今日の記録のみを取り消します。過去の記録は変更されません。

通知コマンド

/notice
手動通知送信

未服薬のユーザー全員に、公開の通知を送信します。定期通知と同じ内容（メンション、Geminiメッセージ、GIF、カレンダー）が即座に送信されます。

/notice

💡 使用例

今すぐ未服薬者に通知を送信したい場合：

/notice

📋 レスポンス例

通知を送信した場合：「未服薬のユーザーに通知を送信しました。」
全員服薬済みの場合：「全員が服薬済みのため、通知は送信しませんでした。」

ℹ️ 情報: コマンドの実行結果は、実行者本人にのみ表示されます。他のユーザーには見えません。
/test_notice
テスト通知送信

コマンド実行者本人にのみ、テスト通知を送信します。通知がどのように表示されるかを確認するのに便利です。

/test_notice

💡 使用例

通知の表示を確認したい場合：

/test_notice

📋 レスポンス例

「【テスト通知】」というタイトルで、通常の通知と同じ内容が送信されます。カレンダーも表示されます。

🔒 プライベート: このコマンドの結果は他の人には見えません。あなただけに送信されます。

コマンドの詳細情報

コマンドの使い方

  • すべてのコマンドはスラッシュ（/）で始まります
  • コマンド入力時にTabキーでオートコンプリートが利用できます
  • ユーザー指定時は@メンションを使用してください
  • コマンドの実行には適切な権限が必要です

カレンダーの見方

  • ✅ 服薬済みの日
  • 🗓️ 今日の日付
  • ⚠️ 服薬忘れの日（過去）
  • 空白 未来の日付

注意事項

  • コマンドはサーバー管理者または適切な権限を持つユーザーのみが実行できます
  • 服薬記録は日付単位で管理されます
  • 過去の記録は手動で変更できません
  • BOTがオフラインの場合はコマンドが実行できません
